# ProMed Technologies - Logistics Module Knowledge Base

## Overview
The Logistics module manages all aspects of delivery operations including tripsheets, loads, drivers, vehicles, fleet tracking, fuel management (TFN), customer management, invoices, sleep outs, vehicle maintenance, scrap records, partial deliveries, order cancellations, and route optimization.

Access the Logistics module from the main navigation. The module has multiple tabs:
- Dashboard, TripSheets, Loads, Invoices, Drivers, Vehicles, Fleet Tracking, Warehouses, Customers, TFN (Fuel), Sleep Outs, Maintenance, Reports, Scrap, Part Delivered, Cancellations

## TripSheets

### What is a TripSheet?
A TripSheet (TS-XXXXXX) is a document that groups multiple delivery invoices for a single trip. It contains:
- Driver assignment
- Vehicle assignment
- Starting and ending warehouse
- List of delivery stops (invoices)
- KM readings (start/end)
- Status tracking
- Commodities and quantities per stop

### TripSheet Statuses
1. **Available**: Created but not yet assigned to a driver/vehicle
2. **Active**: Currently in progress (driver is out on deliveries)
3. **Completed**: All deliveries finished, trip is done
4. **Cancelled**: TripSheet was cancelled

### TripSheet Workflow
1. Import invoices from SYSPRO ERP (JSON or Excel format)
2. Create tripsheet from pending invoices (can group by route/province/area)
3. Assign driver and vehicle
4. Generate tripsheet PDF for printing or emailing
5. Email tripsheet to warehouse manager for dispatch
6. Driver activates tripsheet when leaving warehouse
7. Driver completes deliveries and captures POD (Proof of Delivery)
8. Mark tripsheet as complete when all stops delivered

### TripSheet Actions
- Create from invoices (auto-groups by area)
- Assign driver and vehicle
- Activate (start trip)
- Update status
- Generate PDF
- Email to driver or warehouse manager
- Import from Excel (preview and commit)
- Delete

### POD (Proof of Delivery) Status Colors
- **Green**: POD uploaded/captured successfully
- **Orange/Amber**: Partial delivery (some items not delivered)
- **Red**: No POD captured yet

## Loads

### What is a Load?
A Load (LD-XXXXXX) represents a delivery assignment with:
- Customer information
- Origin and destination
- Assigned vehicle and driver
- Scheduled delivery date
- Delivery status
- Stops with customer addresses and commodities
- Line items with invoice details

### Load Statuses
- **Pending**: Created, awaiting assignment
- **Assigned**: Driver and vehicle assigned
- **In Transit**: Currently being delivered
- **Delivered**: Successfully completed
- **Cancelled**: Load was cancelled

### Load Features
- Create loads with multiple stops and items
- Assign and reassign drivers/vehicles
- Upload proof of delivery (photo/document)
- Download POD files
- Sync invoice statuses with load statuses
- Address issue detection for load stops
- AI address suggestions for incomplete addresses
- Bulk fix stop addresses from customer data
- Bulk fix stop provinces

### Address Issues
Customers imported from ERP may have incomplete address information:

#### Types of Address Issues
1. **No Address**: Missing street address
2. **No City**: Missing city name
3. **No Province**: Missing province
4. **No GPS**: Missing latitude/longitude coordinates

#### Fixing Address Issues
1. Click "Address Issues" card on dashboard (shows count)
2. Select issue type tab
3. Click edit button on customer
4. Fill in missing information (use AI suggestions if available)
5. Save changes

#### AI Address Tools
- **Address Suggestions**: AI generates address corrections
- **Apply Suggestion**: One-click apply AI suggestions
- **Cleanup Addresses**: Batch AI address standardization
- **Map Provinces**: Auto-map provinces from address text
- **Map Provinces (Gemini)**: Use Google Gemini AI for province mapping
- **Match by Names**: Match customers by similar names
- **Match Addresses AI**: AI-powered address matching across records
- **Sync from Tripsheets**: Sync addresses from existing tripsheet data
- **Address Quality Stats**: View data quality metrics

### Customer Contracts
Track contracts per customer:
- Contract terms and conditions

- Create and view contracts

## TFN (TransFleet Network) - Fuel Management

### What is TFN?
TFN is the fuel card system used for vehicle refueling. The integration allows:
- View and sync fuel transactions
- Monitor fuel card balances
- Track fuel consumption per vehicle
- Find nearest fuel depots
- Manage fuel orders

### TFN Features
- **Fuel Transactions**: All fuel purchases by vehicle (filterable by vehicle, date, type)
- **Fuel Efficiency**: Fuel efficiency data analysis per vehicle
- **TFN Vehicles**: View vehicles registered with TFN
- **Account Balances**: Current fuel card account balances
- **Fuel Orders**: Pending fuel authorization requests
- **Fuel Depots**: Map of TFN depot locations nationwide
- **Nearest Depot**: Find closest fuel depot to any location
- **Dashboard Summary**: TFN overview metrics

### TFN Sync
- Sync vehicles from TFN
- Sync fuel transactions from TFN
- Full sync (both vehicles and transactions)

## Sleep Outs (Driver Overnight Allowance)

### What is a Sleep Out?
When drivers must stay overnight away from home, they receive a food allowance (typically R250).

### Sleep Out Process
1. Request submitted (usually by dispatcher or logistics coordinator)
2. Manager reviews the request
3. Approve or reject the sleep out
4. Once approved, marked for payment processing
5. Finance processes payment
6. Marked as paid

### Sleep Out Statuses
- **Requested**: Pending approval
- **Approved**: Approved, waiting for payment
- **Rejected**: Request denied
- **Paid**: Payment processed

### Sleep Out Features
- Filter by driver, status, date range, company
- View summary statistics
- Create, edit, delete sleep out records
- Approve and mark as paid workflows

## Common Questions

Q: How do I create a tripsheet?
A: Go to TripSheets tab > Click "Create TripSheet" > Select pending invoices > Assign driver/vehicle > Save

Q: How do I track a vehicle?
A: Go to Fleet Tracking tab > Select vehicle from list > View on map in real-time

Q: How do I approve a sleep out?
A: Click Sleep Outs tab > Find pending request > Click Approve or Reject

Q: How do I fix a customer's address?
A: Click Address Issues on dashboard > Find customer > Edit > Fill in details (or use AI suggestions) > Save

Q: How do I see fuel consumption?
A: Go to TFN tab > View transactions and fuel efficiency data

Q: How do I import invoices?
A: Go to Invoices tab > Click Import > Select JSON or Excel file > Review > Confirm

Q: How do I record a partial delivery?
A: Go to Part Delivered tab > Select invoice > Record delivered quantity

Q: How do I cancel an order?
A: Go to Cancellations tab > Create cancellation > Enter reason > Submit for approval

Q: How do I schedule vehicle maintenance?
A: Go to Maintenance tab > Click create > Select vehicle and type > Set schedule

Q: How do I optimize delivery routes?
A: The system uses Google Maps to optimize routes. Use "Optimize Load" on any load for suggested route order.
